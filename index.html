Mapa mental — Proceso de Innovación

Este mapa mental representa de forma estructurada las fases del proceso de innovación y su relación directa con la Administración de Proyectos. Cada etapa requiere planificación, control de recursos, gestión de riesgos y evaluación continua para asegurar resultados viables y sostenibles dentro de un entorno organizacional.

Cuadro comparativo — Metodologías de Administración de Proyectos

Metodología Tipo Fases / Procesos Ventajas Limitaciones Ejemplo de aplicación
PMBOK (PMI) Tradicional Inicio, planificación, ejecución, monitoreo y cierre Alta estandarización y control Menor flexibilidad ante cambios Proyectos de infraestructura
Waterfall Tradicional Fases secuenciales Claridad en etapas No admite cambios tardíos Construcción
PRINCE2 Tradicional Procesos controlados por etapas Enfoque en negocio Documentación extensa Gobierno y sector público
Scrum Ágil Sprints iterativos Alta adaptabilidad Requiere compromiso del equipo Desarrollo de software
Extreme Programming (XP) Ágil Iteraciones cortas y mejora continua Calidad del producto No apto para proyectos grandes Aplicaciones web
Design Thinking Ágil Empatizar, definir, idear, prototipar y probar Enfoque centrado en el usuario No siempre estructurado Innovación de productos
